Follow-Up Problems

Interviewers rarely ask a brand-new problem — they twist one you know. This capstone classifies the five ways a follow-up transforms a base problem, so you can recognize the twist and reuse the core idea.

Five kinds of follow-up

1 · 1-D → 2-D

Try the same idea one dimension up. Find Peak I→II, Trapping Water I→II, Subarray Sum → Submatrix Sum.

2 · Array → circular array

Apply the circular-array trick. Continuous Subarray Sum.

3 · Strengthen the condition

The method may have to change entirely. Wiggle Sort I → II.

4 · Same essence, new costume

Reworded but identical. Number of Airplanes ≡ Meeting Rooms; many Backpack variants.

5 · Different description, same method

Needs analysis to see through. Forward-pointer problems; Quick Sort ≡ Nuts & Bolts.

The skill Each pattern is a lens: when a question feels new, ask “is this a dimension bump? a circular twist? a strengthened constraint? a re-skin? or the same method in disguise?” Naming the transformation usually hands you the solution.
Pattern 1 & 2 in action

Subarray Sum & its follow-ups

Base: Subarray Sum (find a zero-sum range) uses a prefix-sum → index hash map. Two follow-ups:

  • Submatrix Sum (1-D→2-D): fix a pair of rows, collapse the band between them into a 1-D column-sum array, then it's exactly the 1-D zero-sum subarray problem — O(rows²·cols).
  • Continuous Subarray Sum II (array→circular): the best circular subarray is either a normal max subarray, or total − (min subarray) when it wraps.
Python · max circular subarray 
def max_circular_subarray(nums):
    def kadane(a, sign):                # sign=+1 for max, -1 for min
        best = cur = a[0] * sign
        for x in a[1:]:
            x *= sign
            cur = max(x, cur + x)
            best = max(best, cur)
        return best * sign
    total = sum(nums)
    max_normal = kadane(nums, 1)
    min_sub = kadane(nums, -1)
    if total == min_sub:                  # all negative → wrap would be empty
        return max_normal
    return max(max_normal, total - min_sub)

Partition follow-ups

Quick Select vs a heap — pick by the question
Heap / PriorityQueueQuick Select
TimeO(n log k)O(n) average
Best forTop-K (you want all k)the kth element alone

Nuts & Bolts (Quick Sort in disguise): you can only compare a nut to a bolt, never nut-to-nut. Use one bolt to partition the nuts, then the matching nut to partition the bolts — quicksort with cross-comparisons.

Wiggle Sort (strengthened condition): want a[0] ≤ a[1] ≥ a[2] ≤ …. One greedy pass — fix each adjacent pair that violates the up/down rule:

Python · Wiggle Sort 
def wiggle_sort(nums):
    for i in range(len(nums) - 1):
        if (i % 2 == 0) == (nums[i] > nums[i + 1]):   # wrong direction
            nums[i], nums[i + 1] = nums[i + 1], nums[i]

Wiggle Sort II (strict </>, duplicates allowed) needs a different method: find the median (Quick Select), then place numbers at interleaved indices — a genuine method change, not a tweak.

A reusable design pattern

Iterator problems

The iterator template (3 rules) 1. Turn the structure into a stack. 2. Put the real logic in has_next (advance lazily to the next valid element). 3. next just does one pop. This keeps next/has_next consistent and idempotent.
Python · BST Iterator (in-order, O(h) space) 
class BSTIterator:
    def __init__(self, root):
        self.stack = []
        self._push_left(root)

    def _push_left(self, node):
        while node:
            self.stack.append(node)
            node = node.left

    def has_next(self):
        return len(self.stack) > 0

    def next(self):
        node = self.stack.pop()
        self._push_left(node.right)        # next smallest is leftmost of right subtree
        return node

Conclusion

The Follow-Up checklist

  1. Classify the twist: 1-D→2-D, circular, stronger condition, re-skin, or same method, new description.
  2. Subarray sum ⇒ prefix-sum hashmap; 2-D collapses to 1-D by fixing a row band; circular = max(normal, total − minSub).
  3. kth element ⇒ Quick Select O(n); all of Top-K ⇒ heap O(n log k).
  4. Nuts & Bolts = quicksort with cross-comparisons; Wiggle Sort II = median + interleave.
  5. Iterators: stack + logic in has_next + next pops once.
