Two Pointers

Two indices that sweep an array, never backing up — so the whole scan is O(n). Two shapes cover everything: pointers moving the same direction, and pointers moving toward each other. They power the entire partition and Two-Sum problem families.

The two shapes

Same-direction vs opposite-direction

Same-direction

Both pointers move forward; one lags behind the other (a “fast / slow” pair). Used for in-place compaction. e.g. Move Zeroes, Remove Duplicates.

Opposite-direction

One starts at the head, one at the tail; they walk inward until they meet. e.g. Valid Palindrome, partition, Two Sum on a sorted array.

Why it's always O(n) Neither pointer ever looks back. Opposite-direction: the two pointers split the work and meet in the middle — total distance ≈ n. Same-direction: both march from left to right exactly once. Either way the combined travel is linear.
Python · Valid Palindrome (opposite-direction) 
def is_palindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1
        if s[left].lower() != s[right].lower():
            return False
        left += 1
        right -= 1
    return True
Python · Move Zeroes (same-direction) 
def move_zeroes(nums):
    j = 0                          # slow pointer: next slot for a non-zero
    for i in range(len(nums)):     # fast pointer
        if nums[i] != 0:
            nums[j], nums[i] = nums[i], nums[j]
            j += 1
Family 1 · Partition

Partition Array

Rearrange so that every element < k comes before every element ≥ k. The opposite-direction template: grow a “small” region from the left and a “big” region from the right, swapping the pair that's out of place.

Python 
def partition_array(nums, k):
    left, right = 0, len(nums) - 1
    while left <= right:
        while left <= right and nums[left] < k:    # left side already small → skip
            left += 1
        while left <= right and nums[right] >= k:   # right side already big → skip
            right -= 1
        if left <= right:                       # both are misplaced → swap
            nums[left], nums[right] = nums[right], nums[left]
            left += 1
            right -= 1
    return left                                # index = count of elements < k

Variants that reuse this exact skeleton: Partition Array by Odd and Even, Interleaving Positive and Negative Numbers, Sort Letters by Case.

Partition's killer app

Quick Select — kth element in O(n)

Find the kth-smallest without fully sorting. Partition around a pivot; the pivot lands at its final sorted position, so you recurse into only the side that contains k — halving the work each time for O(n) average (vs O(n log n) for a full sort).

Python · kth smallest (k is 1-based) 
def quick_select(nums, k):
    def helper(lo, hi, k):
        if lo == hi:
            return nums[lo]
        pivot = nums[(lo + hi) // 2]
        left, right = lo, hi
        while left <= right:
            while left <= right and nums[left] < pivot:
                left += 1
            while left <= right and nums[right] > pivot:
                right -= 1
            if left <= right:
                nums[left], nums[right] = nums[right], nums[left]
                left += 1
                right -= 1
        # recurse into the single side that holds the kth element
        if lo + k - 1 <= right:
            return helper(lo, right, k)
        if lo + k - 1 >= left:
            return helper(left, hi, k - (left - lo))
        return nums[right + 1]
    return helper(0, len(nums) - 1, k)
Quick Select vs Quick Sort Same partition core; the difference is Quick Sort recurses into both halves (O(n log n)) while Quick Select recurses into one (O(n) average). It's the engine behind “kth largest element” problems.
Three-way partition

Sort Colors & Rainbow Sort

Sort Colors partitions into three regions (0s, 1s, 2s) in one pass — the Dutch national flag. Three pointers: zero and two bound the known regions, i scans.

Python · Sort Colors (Dutch flag) 
def sort_colors(nums):
    zero, i, two = 0, 0, len(nums) - 1
    while i <= two:
        if nums[i] == 0:
            nums[i], nums[zero] = nums[zero], nums[i]
            zero += 1
            i += 1
        elif nums[i] == 2:
            nums[i], nums[two] = nums[two], nums[i]
            two -= 1                    # don't advance i: swapped-in value unexamined
        else:
            i += 1
Rainbow Sort (Sort Colors II) — k colors For k colors the optimal is not O(nk) counting — it's O(n log k): recursively partition the color range in half (split around the middle color value), and within each half partition the elements, then recurse. Each element moves O(log k) times. The deck's quiz answer to “guess the optimal complexity” is O(n log k).

(The deck also name-drops joke sorts — pancake, sleep, spaghetti, bogo — but pancake sort can genuinely appear in interviews.)

Family 2 · Two Sum

HashMap vs Two Pointers

The Two Sum family is the other half of the chapter. Two engines solve it, and which one fits depends on the exact variant:

VariantBest toolWhy
Array unsorted, return indicesHashMap O(n)One pass, remember complements
Array already sortedTwo Pointers O(n)O(1) extra space, no hashing
Two Sum — data structure designHashMap onlyadd/find online forces hashing
Python · unsorted (HashMap) 
def two_sum(nums, target):
    seen = {}                          # value -> index
    for i, x in enumerate(nums):
        if target - x in seen:
            return [seen[target - x], i]
        seen[x] = i
Python · sorted (Two Pointers) 
def two_sum_sorted(nums, target):
    left, right = 0, len(nums) - 1
    while left < right:
        s = nums[left] + nums[right]
        if s == target:
            return [left, right]
        elif s < target:
            left += 1                    # need a bigger sum
        else:
            right -= 1                   # need a smaller sum
“For a problem about how two variables combine, loop one variable and then study how the other one must change.” — the Palm-Breaking Form (the instructor's rule of thumb)

That single idea generates the whole zoo of counting variants — fix/loop one side, slide the other: count pairs ≤ target, count pairs ≥ target, closest to target, unique pairs (dedup by skipping equal values after each hit), and difference = target (a same-direction variant).

The flagship

3Sum

Find all unique triples summing to 0. Apply the Palm-Breaking Form literally: fix one number (loop it), then the rest is a sorted Two-Pointer Two Sum for -nums[i]. Sort first so dedup is just “skip equal neighbors.” Overall O(n²).

Python 
def three_sum(nums):
    nums.sort()
    res, n = [], len(nums)
    for i in range(n - 2):
        if i > 0 and nums[i] == nums[i - 1]:
            continue                       # dedup the fixed element
        left, right = i + 1, n - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s == 0:
                res.append([nums[i], nums[left], nums[right]])
                left += 1
                right -= 1
                while left < right and nums[left] == nums[left - 1]:
                    left += 1            # dedup the pair
                while left < right and nums[right] == nums[right + 1]:
                    right -= 1
            elif s < 0:
                left += 1
            else:
                right -= 1
    return res

Follow-ups all reuse this loop-one / two-pointer-the-rest pattern: 3Sum Closest, 4Sum (loop two, two-pointer the rest → O(n³)).

Conclusion

The Two Pointers checklist

  1. Two flavors: same-direction (fast/slow, compaction) and opposite-direction (head/tail, converge). Both are O(n) — no pointer backtracks.
  2. Partition is the opposite-direction template; Quick Select applies it for kth-element in O(n) average.
  3. Sort Colors = three-way Dutch-flag partition; Rainbow Sort for k colors is O(n log k).
  4. Two Sum: HashMap when unsorted / online, Two Pointers when sorted.
  5. The Palm-Breaking Form: to combine two variables, loop one and slide the other — this generates every Two-Sum variant and 3Sum/4Sum.
  6. Always sort then skip equal neighbors to dedup triples/pairs.
