Linked List & Array

Two workhorses with two signature techniques. Linked lists live and die by the dummy node and fast/slow pointers; arrays by the prefix sum and clean merging. Small tricks, enormous reach.

Four techniques

A sentinel before the head so you never special-case an empty/changing head.

Linked List · technique 1

The Dummy Node

“When do you need a dummy node? Whenever the list's structure changes — because then the head might change. You store the head in dummy.next and return dummy.next at the end. You don't need to delete the dummy; the garbage collector reclaims it.” — the instructor [48:19]

The template is always the same: create dummy, point it at the head, build using a moving pointer, then return dummy.next. Here it removes the “what if the result head differs from the input head?” worry from Merge Two Sorted Lists:

Python 
def merge_two_lists(l1, l2):
    dummy = ListNode(0)                  # sentinel; real head ends up at dummy.next
    tail = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            tail.next, l1 = l1, l1.next
        else:
            tail.next, l2 = l2, l2.next
        tail = tail.next
    tail.next = l1 or l2                  # attach the remaining tail
    return dummy.next

The deck's “eight questions” boil down to: use it when the head may move; one dummy is enough; its value doesn't matter; you needn't delete it. Problems worth doing with it: Partition List (two dummies — a “small” list and a “big” list, then stitch), Reverse Linked List II, Swap Two Nodes, Reorder List, Rotate List, Reverse Nodes in k-Group.

Linked List · technique 2

Fast / Slow Pointers

Advance one pointer twice as fast as the other. Two payoffs in O(1) extra space: find the middle (slow is at the midpoint when fast reaches the end) and detect a cycle (a faster runner laps a slower one only on a circular track).

Python · cycle detection 
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next                # +1
        fast = fast.next.next            # +2
        if slow is fast:               # they meet ⇒ there is a cycle
            return True
    return False
Follow-ups Linked List Cycle II (find the entry node), and Intersection of Two Linked Lists — both are the same two-runner idea. The middle-finder also powers Sort List below.
Putting it together

Sort List — merge sort on a linked list

Merge sort is the natural fit: split with fast/slow, sort each half, then merge — O(n log n) time. (A linked list splits in O(1) with no scratch array, which is why it's preferred over quicksort here.)

Python 
def sort_list(head):
    if not head or not head.next:
        return head
    slow, fast = head, head.next         # split into two halves
    while fast and fast.next:
        slow, fast = slow.next, fast.next.next
    mid = slow.next
    slow.next = None
    return merge_two_lists(sort_list(head), sort_list(mid))
The deck's two quiz questions Which sorts are O(n log n) time? Merge sort, quicksort (average), heap sort. Which sorts are O(1) extra space? Insertion, selection, bubble, and heap sort. Merge-sorting a linked list is the sweet spot of O(n log n) time with no auxiliary array.

Related: Copy List with Random Pointer (HashMap old→new, or interleave-and-split), Convert Sorted List to Balanced BST, Convert BST to Doubly Linked List.

Array · technique 3

Prefix Sum (subarray problems)

Almost every subarray problem wants a prefix sum. Define P[i] = A[0] + A[1] + … + A[i-1] with P[0] = 0 — built in O(n). Then any subarray sum is a single subtraction:

Python · Maximum Subarray (prefix-min) 
def max_subarray(nums):
    prefix, min_prefix, best = 0, 0, float("-inf")
    for x in nums:
        prefix += x
        best = max(best, prefix - min_prefix)   # best subarray ending here
        min_prefix = min(min_prefix, prefix)      # cheapest left edge so far
    return best
Python · Subarray Sum (find a zero-sum range) 
def subarray_sum(nums):
    prefix = 0
    seen = {0: -1}                       # prefix value -> index
    for i, x in enumerate(nums):
        prefix += x
        if prefix in seen:                 # same prefix twice ⇒ zeros in between
            return [seen[prefix] + 1, i]
        seen[prefix] = i

Same toolbox: Window Sum, Subarray Sum Closest (sort the prefix sums, then the closest pair of prefixes gives the closest-to-zero subarray).

Array · technique 4

Sorted Arrays — merging

Two pointers walking two sorted arrays. The neat variant is Merge Sorted Array (merge B into A in place): fill from the back so you never overwrite unread elements of A.

Python · merge B into A (A has trailing space) 
def merge(A, m, B, n):
    i, j, k = m - 1, n - 1, m + n - 1     # write from the largest end
    while i >= 0 and j >= 0:
        if A[i] > B[j]:
            A[k] = A[i]; i -= 1
        else:
            A[k] = B[j]; j -= 1
        k -= 1
    while j >= 0:                          # leftover B (leftover A is already in place)
        A[k] = B[j]; j -= 1; k -= 1
Related Intersection of Two Arrays; sparse dot product — when both arrays are huge but mostly zeros, store only (index, value) pairs and two-pointer the non-zeros; Median of Two Sorted Arrays — the hard one, a binary-search-on-partition that finds the kth element in O(log(m+n)).
Conclusion

The Linked List & Array checklist

  1. Dummy node whenever the list's structure (and thus the head) can change; return dummy.next, no deletion needed.
  2. Fast/slow pointers find the middle and detect cycles in O(1) space.
  3. Sort List = merge sort (split with fast/slow + merge) → O(n log n).
  4. Subarray problems ⇒ prefix sum: Sum(i…j) = P[j+1] − P[i] in O(1).
  5. Max subarray via prefix − running min; zero-sum range via prefix-in-hashmap.
  6. Merge sorted arrays in place by writing from the back.
