Union Find & Trie

Two data structures that solve problems nothing else does cleanly: Union Find answers “are these in the same group?” and merges groups in near-O(1); Trie stores strings by shared prefix for fast prefix queries and grid word-search.

Frame

Pick the structure for the query

A good data structure is the right relationship between data. Use Union Find when the problem is about set membership and merging; use a Trie when it's about string prefixes or matching characters one at a time across a grid.

Structure 1

Union Find (Disjoint Set)

Maintains a forest of disjoint sets. Two operations, both ~O(1) amortized (formally O(log*n), essentially constant): find the representative (root) of an element's set, and union two sets. Two techniques make it fast:

Path compression

During find, point every node on the path straight at the root — so future finds are O(1).

Union by size/rank

Always hang the smaller tree's root under the bigger one. Merge happens between roots — the children never matter.

Python · full template 
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))      # each node is its own root
        self.size = [1] * n
        self.count = n                       # number of disjoint sets

    def find(self, x):                       # with path compression
        root = x
        while self.parent[root] != root:
            root = self.parent[root]
        while self.parent[x] != root:           # point the whole path at root
            self.parent[x], x = root, self.parent[x]
        return root

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return
        if self.size[ra] < self.size[rb]:       # attach smaller under bigger
            ra, rb = rb, ra
        self.parent[rb] = ra                 # merge roots only
        self.size[ra] += self.size[rb]
        self.count -= 1

    def connected(self, a, b):
        return self.find(a) == self.find(b)
The native + derived operations

Connecting Graph I / II / III

n nodes, no edges; you connect (a, b) and answer queries. The three variants show exactly which questions Union Find answers — two native, two derived:

VariantQueryMechanism
Connecting Graphare a and b connected?native — find(a)==find(b)
Connecting Graph IIsize of a's componentderived — keep size[root]
Connecting Graph IIIhow many components total?derived — keep a count
The exam reduction Union Find only really does two things: merge two sets and test same-set. The skill is recognizing how to rephrase the problem into those two — component size and component count are just counters you maintain alongside the merges.
Where it shows up

Union Find applications

  • Number of Islands II — islands appear one at a time; each new land cell unions with its land neighbors, decrementing the component count. (Online version that BFS can't do efficiently.)
  • Graph Valid Tree — n nodes form a tree iff exactly n−1 edges and no union ever joins two already-connected nodes (no cycle), ending with one component — O(n).
  • Surrounded Regions — union all border-connected ‘O’s to a virtual “escape” node; whatever isn't connected to it gets captured.
  • Accounts Merge — union accounts that share any email, then group by root.
Structure 2

Trie (Prefix Tree)

A tree where each edge is a character and each root-to-node path spells a prefix. Shared prefixes share nodes — so prefix queries and character-by-character matching are natural.

Python · Implement Trie 
class TrieNode:
    def __init__(self):
        self.children = {}                 # char -> TrieNode
        self.is_word = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for ch in word:
            node = node.children.setdefault(ch, TrieNode())
        node.is_word = True

    def _find(self, s):
        node = self.root
        for ch in s:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node

    def search(self, word):
        node = self._find(word)
        return node is not None and node.is_word

    def starts_with(self, prefix):
        return self._find(prefix) is not None

Add and Search Word — wildcards

· search supports “.”

When search may contain . (matches any char), the lookup becomes a small DFS over the Trie — branch into all children on a dot.

Python 
def search(self, word):
    def dfs(node, i):
        if i == len(word):
            return node.is_word
        ch = word[i]
        if ch == ".":
            return any(dfs(c, i + 1) for c in node.children.values())
        if ch in node.children:
            return dfs(node.children[ch], i + 1)
        return False
    return dfs(self.root, 0)
Hash vs Trie A hash set finds a whole string in O(L) too — but it can't answer prefix queries or share work across many strings. The Trie wins when you need prefixes, wildcard search, or character-by-character DFS over a grid (e.g. Word Search II: walk the board while walking the Trie, pruning dead prefixes instantly).
Conclusion

The Union Find & Trie checklist

  1. Union Find = near-O(1) “same set?” + “merge sets”, via path compression + union by size (merge roots, ignore children).
  2. Track size[root] and a count for the derived “component size / number of components” queries.
  3. Reach for it on Number of Islands II, Graph Valid Tree, Surrounded Regions, Accounts Merge — rephrase the problem into merge + same-set.
  4. Trie stores strings by shared prefix: O(L) insert/search plus prefix and wildcard queries a hash can't do.
  5. Wildcard search = DFS over the Trie; grid word-search walks board and Trie together.
