Binary Search Deepened & Sweep Line

Three techniques for “better than O(n)” and interval problems: binary search on a value range (not an index), the sweep line for overlapping intervals, and the monotonic deque for sliding-window extremes.

Technique 1

Binary search on a value range

When there's no array to index into, binary-search the answer value — valid whenever feasibility is monotonic (if value v works, everything on one side of v works). Four steps: (1) bound the feasible range, (2) guess the middle, (3) check the condition, (4) shrink toward the boundary.

Python · Find the Duplicate Number (no extra space, no mutation) 
def find_duplicate(nums):
    start, end = 1, len(nums) - 1            # values live in [1, n]
    while start + 1 < end:
        mid = start + (end - start) // 2
        if sum(x <= mid for x in nums) > mid:   # too many ≤ mid ⇒ dup is low
            end = mid
        else:
            start = mid
    return start if sum(x <= start for x in nums) > start else end
Floating-point binary search (Sqrt II) When the answer is a double, you can't stop at adjacent integers — loop while end - start > eps (e.g. 1e-9) instead, then return either bound.
2-D binary search

Find Peak Element II

Find a cell greater than its four neighbors. Binary-search the row range: at the middle row, find its max column; whichever vertical neighbor is bigger tells you which half holds a peak — O(n log n).

Python 
def find_peak_ii(A):
    top, bottom = 0, len(A) - 1
    while top + 1 < bottom:
        mid = (top + bottom) // 2
        col = max(range(len(A[mid])), key=lambda j: A[mid][j])  # max of mid row
        if A[mid][col] < A[mid - 1][col]:
            bottom = mid                  # bigger above ⇒ peak in upper half
        elif A[mid][col] < A[mid + 1][col]:
            top = mid                     # bigger below ⇒ peak in lower half
        else:
            return [mid, col]            # bigger than both ⇒ it's a peak
    # scan the two surviving rows for the final peak
    row = top if max(A[top]) > max(A[bottom]) else bottom
    return [row, max(range(len(A[row])), key=lambda j: A[row][j])]
“One problem sorts candidates into five tiers” The instructor uses Find Peak to rank interviewees: (1) Peak I in O(n) — only writes for-loops; (2) Peak I in O(log n) — can optimize; (3) Peak II in O(n²) — can't generalize; (4) Peak II in O(n log n) — generalizes the 1-D idea; (5) proves Peak II is O(n) — generalizes even further. The goal of the chapter is to move you from tier 1–3 to tier 4–5.
Technique 2

Sweep Line

The signature of an interval problem: events live at interval endpoints. Turn each interval into a start (+1) and an end (−1) event, sort by time, and sweep left to right keeping a running count.

Python · Number of Airplanes in the Sky 
def count_of_airplanes(airplanes):
    events = []
    for a in airplanes:
        events.append((a.start, 1))      # +1 takeoff
        events.append((a.end, -1))       # -1 landing
    events.sort()                          # tie: -1 sorts before +1 → landing first
    count = best = 0
    for _, delta in events:
        count += delta
        best = max(best, count)
    return best

The sweep-line recipe — events are intervals → endpoints are start/end → sort → sweep — also drives Building Outline (the skyline problem, with a heap tracking the current tallest building).

Technique 3

Monotonic Deque

For Sliding Window Maximum, keep a double-ended queue of indices whose values are decreasing. The front is always the window's max; push pops smaller tails (they can never be the max while a bigger, later element exists), and the front is dropped when it leaves the window — O(n) total.

Python 
from collections import deque

def max_sliding_window(nums, k):
    dq = deque()                           # indices; nums[dq] strictly decreasing
    result = []
    for i, x in enumerate(nums):
        while dq and nums[dq[-1]] <= x:    # smaller tails can't be max anymore
            dq.pop()
        dq.append(i)
        if dq[0] <= i - k:                 # front slid out of the window
            dq.popleft()
        if i >= k - 1:
            result.append(nums[dq[0]])    # front = window maximum
    return result
Just master this one The deque section reduces to Sliding Window Maximum: maintain a set of candidate maxima (a monotonic queue). If you own this problem, you own the technique.
Conclusion

The Ch4 checklist

  1. No array to index? Binary-search the value when feasibility is monotonic — bound, guess, check, shrink.
  2. Floating-point answer ⇒ loop until end − start < eps.
  3. Find Peak II = binary search on rows (O(n log n)); aim for the “generalize” tier.
  4. Sweep line: intervals → ±1 endpoint events → sort → sweep a running count (Airplanes, Building Outline).
  5. Monotonic deque for sliding-window max — front holds the max, O(n).
