Follow-Ups & the Sliding-Window Template

The advanced course is about follow-up questions — the “now make it faster / handle this twist” turns of an interview. Chapter 1's thesis: one forward-pointer template cracks a whole family of window problems, because the inner pointer never moves backward.

Framing

One template, many problems

A naïve substring/subarray scan is two nested loops — O(n²). The key observation: for these problems, when the outer (left) pointer advances, the inner (right) pointer doesn't need to go back to where it started. Both pointers only ever move forward, so the total work is O(n). That sweep is a moving window.

“The optimization comes from the two nested for-loops: the outer pointer still walks element by element, but you prove the inner pointer never has to retreat.” — the instructor [26:10]
The pattern

The moving-window template

Python · the generic shape 
j = 0
for i in range(n):                      # i = left edge of the window
    while j < n and window_can_extend(j):    # push right edge as far as allowed
        add(s[j])
        j += 1
    update_answer(i, j)                # window [i, j) is now maximal/valid
    remove(s[i])                       # i is about to advance → drop it

The only per-problem choices are the extend condition, what you track inside the window, and how you score the answer.

Four instantiations

The window problems

Minimum Size Subarray Sum

· shortest window with sum ≥ s
Python 
def minimum_size_subarray_sum(nums, s):
    j, total, best = 0, 0, float("inf")
    for i in range(len(nums)):
        while j < len(nums) and total < s:
            total += nums[j]
            j += 1
        if total >= s:
            best = min(best, j - i)
        total -= nums[i]                   # shrink from the left
    return best if best != float("inf") else 0

Longest Substring Without Repeating Characters

· window + last-seen map
Python 
def longest_unique_substring(s):
    last = {}                              # char -> last index seen
    start, best = 0, 0
    for i, ch in enumerate(s):
        if ch in last and last[ch] >= start:
            start = last[ch] + 1          # jump left edge past the duplicate
        last[ch] = i
        best = max(best, i - start + 1)
    return best

Minimum Window Substring

· smallest window covering all of t
Python 
from collections import Counter

def min_window(s, t):
    need = Counter(t)
    missing = len(t)                      # chars still needed
    left = 0
    best = (float("inf"), 0, 0)
    for right, ch in enumerate(s):
        if need[ch] > 0:
            missing -= 1
        need[ch] -= 1
        while missing == 0:                # window covers t → try to shrink
            if right - left + 1 < best[0]:
                best = (right - left + 1, left, right)
            need[s[left]] += 1
            if need[s[left]] > 0:
                missing += 1
            left += 1
    return s[best[1]:best[2] + 1] if best[0] < float("inf") else ""

Longest Substring with At Most K Distinct Characters

· shrink when distinct > k
Python 
from collections import defaultdict

def longest_k_distinct(s, k):
    count = defaultdict(int)
    left, best = 0, 0
    for right, ch in enumerate(s):
        count[ch] += 1
        while len(count) > k:               # too many distinct → shrink left
            count[s[left]] -= 1
            if count[s[left]] == 0:
                del count[s[left]]
            left += 1
        best = max(best, right - left + 1)
    return best
The other forward-pointer family

Fast / slow pointers

The same “forward-only” idea, but with the two pointers at a fixed speed/gap rather than a condition-driven window: Remove Nth Node From End (gap of n), Find the Middle of a Linked List, and Linked List Cycle I / II. Recognize both as instances of “two indices, neither rewinds.”

Meta-skill

How to actually beat follow-up questions

The common trap “I got it Accepted” or “I copied the solution once” is not learning — that's why people still freeze on a problem they've technically already done. The number of problems grows, but the number of algorithm tags doesn't.
Periodically organize problems by type, and ask three questions
  1. Which category does this belong to?
  2. What do problems in this category have in common?
  3. What is the shared line of thinking?
Four study habits: preview before class, take notes during, reflect afterward, and do the homework. The problems multiply; the ideas don't.
Conclusion

The Sliding-Window checklist

  1. If a brute force is a nested loop where the inner pointer never needs to rewind, it collapses to an O(n) moving window.
  2. Template: outer i = left edge; inner while extends j; score the window; drop s[i] before advancing.
  3. Per problem, pick only the extend condition, the window state, and the scoring.
  4. Fast/slow pointers are the same forward-only idea with a fixed gap (Nth-from-end, middle, cycle).
  5. Beat follow-ups by grouping problems by type and asking: category? commonality? shared thinking?
