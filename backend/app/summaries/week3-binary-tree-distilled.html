Binary Tree & Divide Conquer

Recursion on trees comes in two flavors — traverse (carry the answer down in a parameter) and divide & conquer (return the answer up from the children). Master the second and roughly 90% of binary-tree interview problems fall to one template.

The core idea

Think whole-vs-children

“For any binary-tree problem, think about the relationship between the whole tree's answer and the answers of its left and right children.” — the Spear-Breaking Form (the instructor's rule of thumb)

That single habit is divide & conquer: solve the two subtrees (assume the recursion works), then combine their results into the answer for the current node. Recursion here is simply one way to implement DFS on a tree.

Warm-up

Time-complexity training (tree recurrences)

Reading a recurrence off the recursion tree:

Split one n-problem into two n/2-problems using…RecurrenceTotal
O(n) merge work (e.g. merge sort)T(n) = 2·T(n/2) + O(n)O(n log n)
O(1) work (e.g. a plain tree traversal)T(n) = 2·T(n/2) + O(1)O(n)

The difference between O(n) and O(n log n) is entirely the per-split combine cost — visiting every node once is O(n); doing linear work at each of log n levels is O(n log n).

Foundations

Traversal orders

Three depth-first orders, distinguished only by when you visit the root relative to its subtrees:

Python · must-memorize iterative inorder 
def inorder(root):
    stack, result, node = [], [], root
    while stack or node:
        while node:                    # dive to the leftmost node
            stack.append(node)
            node = node.left
        node = stack.pop()
        result.append(node.val)         # visit root after its left subtree
        node = node.right
    return result

The course flags the non-recursive preorder and inorder traversals as must-memorize programs.

The big distinction

Traverse vs Divide & Conquer

Both are recursion — the difference is where the answer lives:

TraverseDivide & Conquer
Answer lives in…a parameter carried downthe return value passed up
Directiontop-downbottom-up
Return typeusually void (mutates shared state)a meaningful value
Feels like“walk and record”merge sort / quick sort
“In traverse, 99% of the time you return nothing — you operate on a global. In divide & conquer you almost always return a value with a specific meaning: solve the left subtree, solve the right subtree, then merge.” — the instructor [30:03]

Maximum Depth — both ways

Python · divide & conquer (answer in return value) 
def max_depth(root):
    if not root:
        return 0
    left = max_depth(root.left)         # solve children, trust the recursion
    right = max_depth(root.right)
    return max(left, right) + 1        # merge
Python · traverse (answer in a shared variable) 
def max_depth(root):
    best = [0]
    def traverse(node, depth):
        if not node:
            return
        best[0] = max(best[0], depth)   # record into shared state
        traverse(node.left, depth + 1)
        traverse(node.right, depth + 1)
    traverse(root, 1)
    return best[0]
Default to divide & conquer Prefer the return-value style: no shared mutable state, naturally parallel-looking, and it generalizes to every “relate whole to children” problem — Binary Tree Paths, Minimum Subtree, Path Sum, Flatten Binary Tree to Linked List, Longest Consecutive Sequence, Lowest Common Ancestor.
Python · Lowest Common Ancestor (classic D&C) 
def lowest_common_ancestor(root, a, b):
    if not root or root is a or root is b:
        return root
    left = lowest_common_ancestor(root.left, a, b)
    right = lowest_common_ancestor(root.right, a, b)
    if left and right:                 # a, b split across the two sides → root is the LCA
        return root
    return left or right             # otherwise bubble up whichever side found one
When one return value isn't enough

ResultType

Sometimes the merge needs several facts from each subtree. Return a small bundle (a tuple, or a tiny class) instead of a single number. Canonical case: Balanced Binary Tree needs both “is it balanced?” and “how tall is it?” in one pass.

Python · Balanced Binary Tree — return (is_balanced, height) 
def is_balanced(root):
    def helper(node):
        if not node:
            return True, 0                # (balanced?, height)
        lb, lh = helper(node.left)
        rb, rh = helper(node.right)
        balanced = lb and rb and abs(lh - rh) <= 1
        return balanced, max(lh, rh) + 1
    return helper(root)[0]

Same pattern for Subtree with Maximum Average (carry (sum, count) up so the average is computable at each node in O(1)).

A special tree

Binary Search Tree

Definition: every node's left subtree is smaller, its right subtree is not smaller. The defining consequence:

The BST ⇔ inorder property A BST's in-order traversal is a non-decreasing sequence. So if a tree's inorder isn't non-decreasing, it's definitely not a BST — but the converse is not guaranteed (a tree of all equal values can be non-decreasing in-order without being a valid BST). Use it as a fast necessary check, and operations run in O(h) (tree height) for find / insert / delete.
Python · Validate BST (divide & conquer with bounds) 
def is_valid_bst(root):
    def helper(node, lo, hi):
        if not node:
            return True
        if not (lo < node.val < hi):       # node must fit its allowed range
            return False
        return (helper(node.left, lo, node.val) and
                helper(node.right, node.val, hi))
    return helper(root, float("-inf"), float("inf"))

Other BST staples: Convert BST to Doubly Linked List, BST Iterator, In-order Successor, Search Range, and Insert / Remove Node (removal is the fiddly one — replace with the in-order successor).

Conclusion

The Binary Tree checklist

  1. Compute tree complexity from the recurrence: +O(n)/level ⇒ O(n log n), +O(1)/level ⇒ O(n).
  2. Recursion is one implementation of DFS; tree DFS comes as traverse or divide & conquer.
  3. Traverse = answer in a parameter (void, mutates state); D&C = answer in the return value (merge children). Prefer D&C.
  4. The Spear-Breaking Form: relate the whole tree's answer to its children's.
  5. Need multiple facts per subtree ⇒ a ResultType bundle (Balanced Tree, Max-Average).
  6. BST: inorder is non-decreasing; find/insert/delete in O(h).
  7. Memorize the non-recursive preorder & inorder traversals.
